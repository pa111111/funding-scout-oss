Связки в этом домене не однородны — у каждого типа разная динамика, ёмкость, период удержания и набор сигналов. Источник: посты автора fundoor.pro (@everybodycandoit на Telegram, "My dad works at Hyperliquid") за ноябрь 2025 — март 2026.

1. **Cross-DEX same ticker** — лонг на A, шорт на B по одному и тому же активу. Базовая дельта-нейтральная схема. Ёмкость средняя, удержание часы–сутки. Под-вариант **spot+perp с retroactivity-мультипликатором**: спотовая нога стейкается в протоколе и фармит airdrop поинты (помимо хеджа), perp-нога собирает фандинг. Кейс из стрима — Virtuals: спот стейкался, шорт держал permanent позицию → две доходности на одну дельту. Применимо для активов, у которых spot-leg сама по себе доходна (LST, points-fаrm tokens, governance).

2. **Equity perps weekend** — лонг/шорт по PLTR, NVDA, TSLA и т.п. между Lighter / Pacifica / EdgeX в выходные. Пока NYSE закрыт, цены на DEX расходятся; к открытию сходятся. Funding часто 100–400% APR на стороне с дисбалансом. Ёмкость низкая (~$30k на связку), окно — выходные.

3. **Single-venue correlated pair** — два коррелированных тикера на одной бирже с расходящимся funding (пример: Tesla-сетап 13.11.2025, спред 70–100% APR). Рыночный риск гасится корреляцией, не нужен трансфер между биржами. Ограничение — изолированная маржа (нет переливания). Многократное умножение через **wrapper-биржи** (Based.one = обёртка над Hyperliquid): одна и та же ликвидность, разный funding на фронтах.

4. **Funding sniping** — заход за минуту до фиксации funding и выход через минуты после. Работает на биржах с **редким расчётом funding** (раз в 4–8ч вместо часа), например Carbon. Пример 15.01.2026: Caito short Carbon −1200% / long Paradex +3000%. Ёмкость зависит от OI; удержание минуты. **Вне MVP** — под ручное исполнение и $5k на связку (см. execution.md) физически не успеть руками. Тип сохранён в таксономии для контекста; вернётся при наличии executor-блока.

5. **Long-tail capacity carry** — заход в связку и удержание неделями (пример: NMR на Variational с 03.11.2025, держал >11 дней). Ограничение — **OI лимит на сторону**, часто не дает зайти. Удержание недели–месяц.

6. **Cash-and-carry (spot + short perp) — DEX-only вариант, узкий**. На CEX это массовая схема с unified margin, но funding-scout = DEX-only (см. concept.md). На DEX схема жизнеспособна **только там, где спот и перп на одном venue под единой маржой**: реально это **Hyperliquid (HyperCore)**, изредка пара других интегрированных площадок. На большинстве DEX спот и перп — разные протоколы, что ломает экономику (gas costs на каждую ногу, плохая ликвидность спота, нет cross-margin). APR скромнее остальных типов (10–20% годовых на крупных активах), зато ёмкость самая большая. В постах автора канала этой схемы нет — он гоняет более высокодоходные cross-DEX/weekend связки. В функциональности funding-scout — низкий приоритет: показывать только если пользователь явно фильтрует "high-capacity, low-APR".

**Why:** каждый тип требует своих сигналов и своей логики ранжирования. Топ-list по funding APR смешает их в кашу — sniping-связку с минутным окном не имеет смысла показывать в одном списке с многонедельным carry. У типов разная capacity и разный alpha decay (см. ниже).

**How to apply:**
- В data-model тип связки = first-class field, фильтры в UI по нему.
- Для type 2 (weekend equity) — flag по календарю NYSE; алерт за час до закрытия / открытия.
- Для type 3 — детектор коррелированных пар на одной venue (TSLA + tesla-related, ETH + ETH-LST, и т.п.).
- Для type 4 — отдельный фид по биржам с не-часовым funding interval; алерт за N минут до фиксации.
- Для type 5 — трекинг OI utilization по сторонам, чтобы понять, есть ли слот.
- Alpha decay для всех типов — 2–4 недели после появления новой DEX, потом MM разъедают расхождения. Поэтому **автодетект новых DEX-перпов** обязателен, иначе сканер устаревает.
